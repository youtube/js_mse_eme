You can use ↑↓→← to move around and use ENTER to select.

You can increase the test timeout or download the test package and host it locally to avoid any issues caused by bandwidth and latency.

You can add GET parameters on 2016.html, tip.html, etc to alter the way the tests runs.

  • Use 2016.html?command=run to run all tests.
  • Use 2016.html?tests=1,2,5,10 to select tests 1, 2, 5 and 10.
  • Use 2016.html?exclude=2,5,8 to exclude tests 2, 5 and 8.
  • Use 2016.html?loop=true to turn on loop or tip.html?loop=10 to run the test a certain amount of times.
  • Use 2016.html?stoponfailure=true to turn on stoponfailure.
  • Use 2016.html?disable_log=true to disable log.
  • Use 2016.html?timeout=60000 to set timeout to 60000ms. By default timeout is 30000.
  • Use 2016.html?enablewebm=false to run the tests using mp4/h264. By default the tests run with webm/vp9 for the latest test suites.

Note: Most of these parameters can be combined. So 2016.html?command=run&tests_mask=10101&stoponfailure=true will run tests 1, 3 and 5 automatically and stop on the first failure.
