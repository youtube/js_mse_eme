You can use ↑↓→← to move around and use ENTER to select.

You can increase the test timeout or download the test package and host it locally to avoid any issues caused by bandwidth and latency.

You can add GET parameters on 2017.html, tip.html, etc to alter the way the tests runs.

  • Use 2017.html?command=run to run all tests.
  • Use 2017.html?tests=1,2,5,10 to select tests 1, 2, 5 and 10.
  • Use 2017.html?exclude=2,5,8 to exclude tests 2, 5 and 8.
  • Use 2017.html?loop=true to turn on loop.
  • Use 2017.html?stoponfailure=true to turn on stoponfailure.
  • Use 2017.html?disable_log=true to disable log.
  • Use 2017.html?timeout=60000 to set timeout to 60000ms. By default timeout is 30000.

Note: Most of these parameters can be combined. So 2017.html?command=run&tests_mask=10101&stoponfailure=true will run tests 1, 3 and 5 automatically and stop on the first failure.
